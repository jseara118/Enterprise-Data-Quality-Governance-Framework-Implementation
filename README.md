# Enterprise Data Quality & Governance Framework Implementation

This project demonstrates the design and implementation of an enterprise-level Data Quality and Data Governance framework applied to a simulated production environment.

The solution addresses unreliable data generated by an application and consumed by BI and downstream systems, showcasing how governance, validation, and monitoring can be implemented end-to-end.

---

## 📌 Key Components

- Data Governance Framework (Naming Standards, Ownership, Policies)
- Data Quality Test Plan (Validation strategy across layers)
- Data Quality Rules Catalog (DQ rules and severity model)
- Data Quality Scoring (DQS) & Monitoring
- End-to-End Pipeline Simulation (Raw → Staging → Curated)
- SQL + Python + Databricks implementation

## 🔁 Pipeline Flow

Setup → Ingestion → Validation → Staging → Curated → Monitoring


- **Setup:** Create schemas and tables aligned with governance standards  
- **Ingestion:** Load intentionally dirty data (duplicates, nulls, invalid relationships)  
- **Validation:** Apply Data Quality rules across multiple dimensions  
- **Staging:** Clean and standardize data  
- **Curated:** Produce business-ready datasets  
- **Monitoring:** Track Data Quality Score (DQS) and metrics  

---

## 🧪 Data Quality Framework

The validation framework covers:

- Completeness  
- Uniqueness  
- Referential Integrity  
- Validity  
- Consistency  
- Timeliness  

Each rule is classified by severity (Critical, High, Medium, Low) and linked to governance standards.

## 🎯 Purpose

---

## 🏗️ Architecture Overview

The solution follows a layered data architecture:

- **Raw Layer:** Ingested data with quality issues  
- **Staging Layer:** Data cleaning and standardization  
- **Curated Layer:** Business-ready datasets  

Data Quality validations are applied across all layers to ensure reliability.
